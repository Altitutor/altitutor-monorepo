# Marketing Web Migration

`apps/marketing-web` is the Next.js replacement for the WordPress site currently served from `altitutor.com`.

## SEO Baseline

The first migration slice preserves the current WordPress URL surface before visual redesign work:

- WordPress page content was exported from the public REST API into `apps/marketing-web/src/content/wordpress-pages.json`.
- Per-page metadata is generated from Yoast fields in that export.
- WordPress upload assets referenced by the migrated pages are copied into `apps/marketing-web/public/wp-content/uploads` so existing `/wp-content/uploads/...` URLs continue to work after the domain moves.
- `robots.txt` and `sitemap.xml` are generated by the app.
- Legacy WordPress sitemap URLs redirect to `/sitemap.xml`.
- Retired WordPress registration routes redirect to the relevant current destination.
- Retired WooCommerce/session routes redirect to the student trial booking flow.

Google Search Console export captured on 2026-05-30 showed the highest root-domain traffic on:

- `/`
- `/about/`
- `/classes/ucatprep/`
- `/classes/english-assignment-drafting/`
- `/classes/examprep/`
- `/classes/`
- `/about/contact/`
- `/classes/weekly-classes/`
- `/classes/medical-interview-preparation/`

Treat those as priority pages during any redesign: keep their URL, canonical URL, primary search intent, and substantive copy stable unless there is a deliberate replacement plan.

## Launch Checklist

- Deploy `marketing-web` to Vercel with `altitutor.com` and `www.altitutor.com` configured.
- Confirm staging deployments are not indexable before production launch.
- Before DNS cutover, crawl the current WordPress site and the Vercel preview and compare statuses, canonicals, titles, descriptions, robots directives, sitemap membership, and internal links.
- After launch, submit `https://altitutor.com/sitemap.xml` in Search Console.
- Monitor Search Console page indexing and performance daily for the first two weeks.

## Redesign Rule

Redesign is safe after this baseline exists, but page-level redesigns should be reviewed against the Search Console priority list. Redesign visual layout freely; change URL, canonical, title, H1, and core copy only intentionally.
